# 시작이 제일 무서워 미룬이

## 1. 프로젝트 소개
'시작이 제일 무서워 미룬이'는 할 일을 시작하고, 진행하고, 마무리하기 어려운 사용자를 위해,  
현재 상태와 데드라인을 반영한 **5단계 초소형 퀘스트**를 생성하고 이를 나무가 자라는 모습으로 시각화한 것이다.
사용자는 텍스트 또는 음성으로 현재 할 일을 입력할 수 있으며,  
시스템은 이를 바탕으로 지금 당장 실행 가능한 작은 행동 단위로 작업을 나눈다.  
각 단계를 완료할 때마다 나무가 성장하고, XP·코인·스탬프를 획득하게 된다.

## 2. 기획 의도
이 프로젝트는 **시작 장벽을 낮추고, 작은 성공을 누적하게 만드는 것**에 초점을 둔다.
기존 투두 앱은 사용자가 직접 계획을 세워야 하는 경우가 많지만, 이 게임은 사용자의 입력을 바탕으로 LLM이 현재 상태를 해석하고, 즉시 수행 가능한 행동으로 재구성할 수 있다.

즉, 이 게임은  
- 시작이 막막한 상태
- 부담이 큰 상태
- 에너지가 낮은 상태
- 데드라인이 가까운 상태  
등을 고려하여 **맞춤형 시작 퀘스트**를 제공하는 것을 목표로 한다.

## 3. 사용된 기술
### 3-1. LangChain
사용자의 입력을 바탕으로 하기 싫은 일, 현재의 상태, 데드라인, 가능한 시간을 해석한 뒤 다섯 단계의 구체적인 퀘스트를 분해하는 구조를 가지고 있다. 이 때, LangChain의 ChatOpenAI를 활용하여 

- 퀘스트 생성
- 실패 시 더 작은 퀘스트 재생성
- 세션 회고 생성
  
세 가지를 일관된 인터페이스로 관리할 수 있어야 하기 때문에, LangChain은 이를 묶어주는 역할을 한다.
사용자는 항상 같은 표현으로 상태를 말하지 않기 때문에 단순 키워드 매칭이 아닌 LangChain의 임베딩 구조를 통해 사용자 표현에 따라 가장 가까운 상태를 추론하는 방식으로 확장 가능하다.

### 3-2. LangGraph
입력되는 목표에 따라 받아들이기만 하는 것이 아닌,

- 목표 입력
- 현재 상태 해석
- 데드라인 반영
- 5단계 퀘스트 생성
- 단계별로 성패를 기록
- 실패 시 더 작은 행동으로 재조정
- 회고

워크 플로우 기반 구조를 가지기 때문에 LangGraph를 사용하는 것이 자연스럽다.

## 4. 주요 기능

### 4.1 목표 입력
사용자는 텍스트 또는 음성으로
- 하기 싫은 일
- 현재 상태
- 데드라인 정보  
등을 자유롭게 입력할 수 있다.

### 4.2 5단계 퀘스트 생성
입력된 목표를 기반으로 LLM이
- 사용자의 현재 상태를 추론하고
- 5단계의 구체적이고 실행 가능한 퀘스트로 분해한다.

### 4.3 나무 성장형 UI
퀘스트 진행 상황은 밭 위 작물의 성장으로 시각화된다.

- 1단계: 씨앗
- 2단계: 새싹
- 3단계: 줄기/잎
- 4단계: 꽃
- 5단계: 열매 및 수확

이를 통해 사용자는 자신의 진행 상황을 단순한 체크리스트가 아니라  
**성장하는 것을 시각적으로 확인**할 수 있다.
나무는 별도의 초기화 버튼을 누르지 않는 이상 계속 누적된다.

### 4.4 보상 시스템
각 단계를 완료하면 보상이 코인으로써 지급된다.

### 4.5 상점 시스템
획득한 코인을 사용하여
농장을 꾸미는 아이템을 구매할 수 있다.

### 4.6 음성 입력 지원
사용자는 마이크 버튼을 눌러 음성으로 목표를 입력할 수 있으며,  
STT를 통해 텍스트로 변환되어 동일한 흐름으로 처리된다.

---

## 5. 시스템 구조

### 5.1 입력
유저는 해야 할 일과 데드라인을 텍스트를 통해 입력하거나, 말을 하여 음성을 인식하게 한다.

### 5.2 처리
사용자 목표를 분석하고, 어떤 상태인지 추론하고, 제시된 데드라인을 기반으로 어느 정도로 긴급한지를 판단한다.
판단한 바를 기반으로 다섯 단계의 퀘스트를 생성하고 하단에 완료 여부를 체크한다.
유저가 모두 수행하였으면 보상을 지급하, 회고를 생성한다.

### 5.3 출력
이 게임은 챗봇의 메시지를 출력하고, 나무 성장형 UI를 시각화하고, 상점에서 산 것들을 반영한다.
